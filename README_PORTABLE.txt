========================================
  Medi_Q - QRコード来院者管理システム
  ポータブル版 v1.0
========================================

このソフトウェアは医療施設向けのQRコード来院者管理システムです。


■ 必要な環境
----------------------------------------
- OS: Windows 10 / 11（64bit）
- Node.js: 18.0.0 以上
- メモリ: 4GB以上推奨
- ディスク: 500MB以上の空き容量
- Webカメラ（QRコード読み取り用）
- プリンタ（診察券・診察票印刷用）


■ 初回セットアップ
----------------------------------------
1. Node.jsのインストール
   - https://nodejs.org/ から最新のLTS版をダウンロード
   - インストーラーを実行してインストール

2. Medi_Qの起動
   - 「Medi_Q起動.bat」をダブルクリック
   - 初回起動時は自動的に依存パッケージをインストールします
   - 数分かかる場合があります

3. ブラウザが自動起動
   - http://localhost:3000 が開きます
   - 開かない場合は手動でアクセスしてください


■ 使い方
----------------------------------------
【受付画面】
  URL: http://localhost:3000
  - QRコードをカメラでスキャン
  - 患者情報・診察情報が表示されます
  - 診察票を印刷できます

【管理画面】
  URL: http://localhost:3000/admin
  ログイン情報:
    ユーザー名: admin
    パスワード: admin123

  ※初回ログイン後、必ずパスワードを変更してください
    （設定 > パスワード変更）

【主な機能】
  ✓ 患者管理（登録・編集・QRコード発行）
  ✓ 予約管理（カレンダー表示・予約登録）
  ✓ 受付処理（QRコード読み取り・音声案内）
  ✓ マスタ管理（診察科・担当医・待機場所・検査項目）


■ 停止方法
----------------------------------------
- コマンドプロンプトのウィンドウを閉じる
- または Ctrl+C を押して終了


■ データのバックアップ
----------------------------------------
重要なデータは定期的にバックアップしてください。

【バックアップ対象】
  app\prisma\medi_q.db （患者・予約データ）

【バックアップ方法】
  1. Medi_Qを停止
  2. app\prisma\medi_q.db を別の場所にコピー
  3. 日付を付けて保存（例: medi_q_20260118.db）

【復元方法】
  1. Medi_Qを停止
  2. バックアップファイルを app\prisma\medi_q.db に上書き
  3. Medi_Qを起動


■ 音声案内機能（オプション）
----------------------------------------
音声案内を使用する場合は、VOICEVOX Engineが必要です。

1. VOICEVOX Engineのダウンロード
   https://voicevox.hiroshiba.jp/

2. インストール後、VOICEVOX Engineを起動

3. Medi_Qを起動すると自動的に音声案内が有効になります


■ トラブルシューティング
----------------------------------------
【Q1】起動時に「Node.jsがインストールされていません」と表示される
  A: Node.jsをインストールしてください
     https://nodejs.org/

【Q2】ポート3000が既に使用されている
  A: 他のアプリケーションが3000番ポートを使用しています
     そのアプリケーションを終了してから再起動してください

【Q3】データベースエラーが表示される
  A: app\prisma\medi_q.db が破損している可能性があります
     バックアップから復元してください

【Q4】QRコードが読み取れない
  A: Webカメラの接続を確認してください
     ブラウザがカメラへのアクセスを許可しているか確認してください

【Q5】音声案内が再生されない
  A: VOICEVOX Engineが起動しているか確認してください
     http://localhost:50021 にアクセスできるか確認してください


■ システム要件
----------------------------------------
- インターネット接続: 不要（完全オフライン動作）
- セキュリティ: ローカルネットワーク内で使用
- データ保存: SQLite（ローカルファイル）
- 同時利用: 1台のPC（推奨）


■ セキュリティに関する注意
----------------------------------------
⚠️ このシステムは患者の個人情報を取り扱います

【推奨事項】
  ✓ 管理者パスワードを強力なものに変更
  ✓ 定期的にデータをバックアップ
  ✓ PCにウイルス対策ソフトをインストール
  ✓ OSとブラウザを最新の状態に保つ
  ✓ 使用しない時はPCをロックまたはシャットダウン
